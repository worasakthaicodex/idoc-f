# i18n — รองรับหลายภาษา (ออกแบบเพื่อสเกลระดับโลก)

ภาษาเริ่มต้น: **ไทย + อังกฤษ** · ภาษาฐาน (fallback) = **English** · เพิ่มภาษาใหม่ได้โดยไม่แตะโค้ด

## 1) UI strings (ฝั่งหน้าเว็บ)
- ใช้ **react-i18next** (`i18next` + `react-i18next` + `i18next-browser-languagedetector`)
- init ที่ [`src/i18n/index.ts`](src/i18n/index.ts) — โหลด `locales/{th,en}.json`
- ดิกชันนารีแยกตาม namespace ในไฟล์เดียวต่อภาษา: `common`, `auth`, `errors`, … (เพิ่ม `sales`, `hr`, `companies` ตอนย้ายแต่ละหน้า)
- ใช้งาน:
  ```tsx
  const { t } = useTranslation();
  <label>{t("auth.email")}</label>
  t("auth.devHint", { email, password })   // interpolation
  ```
- **กติกา:** ห้ามฮาร์ดโค้ดข้อความใน JSX อีก — ทุกข้อความต้องเป็น key

## 2) ภาษาของผู้ใช้ (จดจำ + สลับ)
- ปุ่มสลับ: [`src/shared/LangSwitcher.tsx`](src/shared/LangSwitcher.tsx) (วางได้ทุก topbar)
- ลำดับเลือกภาษา: `localStorage["idoc.lang"]` → ภาษาเบราว์เซอร์ → fallback `en`
- **ถัดไป:** เก็บ `locale` ราย user (`employee.locale`, `platform_account.locale`) + `default_locale` ระดับบริษัท แล้ว sync เข้า i18n หลัง login

## 3) ข้อความจาก backend = error **code** (สำคัญสุด)
API ไม่ส่งข้อความผูกภาษาอีกต่อไป — ส่ง **machine code + English fallback**:
```json
{ "status": 409, "code": "auth.invalid_credentials",
  "message": "Incorrect email or password" }
```
- backend: `throw new BusinessException("auth.invalid_credentials", "Incorrect email or password")`
  ([BusinessException](../idoc/src/main/java/com/idoc/shared/exception/BusinessException.java) มี `code`; [ApiError](../idoc/src/main/java/com/idoc/shared/web/ApiError.java) ส่ง `code` ออกไป)
- frontend: [`apiFetch`](src/shared/api.ts) โยน `ApiError { code, message }` → แปลด้วย
  `t("errors." + code, { defaultValue: message })`
- คำแปลของ code อยู่ใน `errors.*` ของ dict ทั้งสองภาษา
- **ข้อดี:** backend ภาษาเดียว (English), client ทุกตัว (เว็บ/มือถือ) แปลเอง, ภาษาใหม่ไม่ต้องแตะ backend

### catalog ของ code (ขยายเรื่อย ๆ)
| code | ความหมาย |
|---|---|
| `auth.invalid_credentials` | อีเมล/รหัสผ่านผิด |
| `auth.gmail_not_enabled` | อีเมลไม่ได้เปิด Gmail login |
| `company.not_found` / `company.suspended` | บริษัทไม่พบ/ถูกระงับ |
| `common.validation` / `common.not_found` | validation / ไม่พบข้อมูล |

## 4) Data & formatting
- master data (ชื่อบริษัท/สินค้า) = ภาษาเดียวต่อ tenant; ฟิลด์ที่ต้องหลายภาษาจริง ๆ ค่อยใช้ `jsonb` locale-map (ยังไม่ทำตอนนี้)
- วันที่/ตัวเลข/สกุลเงิน → `Intl.*` ตาม locale (รวมไว้ที่ helper เดียว)
- เผื่อ RTL (อาหรับ) → ใช้ CSS logical properties (`margin-inline`, `padding-inline`) เมื่อเขียน CSS ใหม่
- *multi-currency = งานแยกใหญ่ ทำทีหลัง*

## 5) แผนย้ายทีละส่วน (incremental)
1. ✅ วางโครง i18n + ปุ่มสลับภาษา + error code (รอบนี้)
2. ✅ นำร่องหน้า **Login** (ครบทั้ง UI + error จาก backend)
3. ⏳ ย้าย Home / Companies / HR / Sales ทีละหน้า (เพิ่ม namespace)
4. ⏳ เก็บ `locale` ราย user (schema + sync หลัง login)
5. ⏳ ย้าย error message ที่เหลือใน backend เป็น code + English

## วิธีเพิ่มภาษาใหม่ (เช่น จีน `zh`)
1. สร้าง `src/i18n/locales/zh.json` (ก๊อปจาก en.json แล้วแปล)
2. ลงทะเบียนใน `LANGS` + `resources` ที่ [`src/i18n/index.ts`](src/i18n/index.ts)
3. จบ — ไม่ต้องแตะ backend
